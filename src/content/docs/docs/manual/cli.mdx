如果你还没有安装 OpenDesk，请先阅读 [快速开始](/docs/quickstart/cli) 指南，了解如何安装、配置和首次运行。

## 1. 命令行参数

### 1.1 完整参数列表

```bash
opendesk [workspace] [options]
```

#### 位置参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `workspace` | 路径 | `.` | 工作空间目录路径 |

#### 全局选项

| 选项 | 简写 | 说明 |
|------|------|------|
| `--model <alias>` | - | 指定使用的模型别名 |
| `-c, --command <command>` | - | 执行指定命令后退出（单次模式） |
| `--export-html <path>` | - | 任务完成时导出为 HTML |
| `--export-json <path>` | - | 任务完成时导出为 JSON |
| `--dump-log <path>` | - | 将所有日志输出转储到指定文件 |
| `--headless` | - | 无头模式运行（需配合 -c 使用） |
| `-v, --version` | - | 打印版本号 |

### 1.2 子命令

```bash
# 插件管理
opendesk plugins install <spec>     # 安装插件
opendesk plugins uninstall <id>    # 卸载插件
opendesk plugins list              # 列出已安装插件
opendesk plugins enable <id>       # 启用插件
opendesk plugins disable <id>      # 禁用插件

# 工具管理
opendesk tools list               # 列出所有可用工具
opendesk tools info <name>         # 显示工具详细信息
opendesk tools call <name> [args]  # 调用工具

# 消息渠道
opendesk channels list            # 列出已安装的消息插件
opendesk channels info            # 加载并检查所有已启用的消息插件
```

### 1.3 使用示例

**基本交互模式：**

```bash
# 在当前目录启动
opendesk

# 指定工作空间
opendesk /path/to/project

# 指定模型
opendesk --model gpt-4
```

**单次命令模式（无头）：**

```bash
# 执行单个命令后自动退出
opendesk -c "帮我写一个 README.md"

# 指定模型执行
opendesk -c "解释这段代码" --model gpt-4

# 执行后导出结果
opendesk -c "分析这个日志文件" --export-html result.html
```

### 1.4 交互模式 vs 无头模式

| 特性 | 交互模式 (`opendesk`) | 无头模式 (`opendesk -c "..."`) |
|------|----------------------|-------------------------------|
| TUI 界面 | 有 | 无 |
| 实时输出 | 流式显示 | 块结束批量输出 |
| 交互能力 | 完整 | 无（自动拒绝权限请求）|
| 适用场景 | 日常对话、开发调试 | 脚本自动化、CI/CD |

详见 [6.1 无头模式](#61-无头模式-headless)。

## 2. 交互式 TUI 模式

### 2.1 界面布局

```
┌──────────────────────────────────────────────────────────────┐
│  OpenDesk Banner (工作空间路径)                               │
├──────────────────────────────────────────────────────────────┤
│  任务视图区域 (消息显示)                                      │
├──────────────────────────────────────────────────────────────┤
│  待办事项列表                                                │
├──────────────────────────────────────────────────────────────┤
│  日志面板 (可选)                                             │
├──────────────────────────────────────────────────────────────┤
│  [状态栏: 模型 | Token数 | 运行时间]                          │
│  > 输入区域                                                  │
├──────────────────────────────────────────────────────────────┤
│  [仪表盘栏: 渠道状态 | 通知数 | ...]                         │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 状态栏信息

状态栏显示：

- **模型名称**：当前使用的模型
- **上下文长度**：模型支持的最大 Token 数
- **Token 统计**：当前会话消耗的 Token 数
- **运行时间**：当前任务的执行时长

### 2.3 输入区域

输入区域支持：

- **普通文本输入**：直接输入自然语言
- **斜杠命令**：以 `/` 开头的命令（如 `/help`）
- **消息队列**：输入多条消息自动排队依次执行
- **自动补全**：输入时自动提示命令和参数
- **配置口令导入**：粘贴口令自动应用配置

### 2.4 仪表盘栏

底部仪表盘栏显示渠道状态和通知数。

## 3. 斜杠命令参考

斜杠命令以 `/` 开头，在输入框中输入并回车执行。

### 3.1 基础命令

| 命令 | 说明 | 快捷键 |
|------|------|--------|
| `/new [消息]` | 启动新任务 | `Ctrl+N` |
| `/switch` | 切换到其他任务 | `Ctrl+L` |
| `/help` | 显示快捷键与命令帮助 | - |
| `/exit` / `/quit` | 退出 OpenDesk | - |

### 3.2 任务管理

| 命令 | 说明 |
|------|------|
| `/clear` | 删除所有任务历史 |
| `/stop` | 停止当前正在执行的任务 |
| `/cancel` | 撤销消息队列中的所有待发送消息 |
| `/rename <新名称>` | 重命名当前任务 |
| `/fork` | 复制当前会话创建新会话 |
| `/bookmark add` | 将当前任务添加到书签 |
| `/bookmark list` | 列出所有书签任务 |
| `/bookmark remove` | 移除当前任务的书签 |

### 3.3 导入导出

| 命令 | 说明 |
|------|------|
| `/export html <文件名>` | 导出会话为 HTML |
| `/export json <文件名>` | 导出会话为 JSON |
| `/export skill <技能名> <路径.zip>` | 导出技能为 zip |
| `/import task <json文件路径>` | 从 JSON 文件导入任务 |

### 3.4 视图与显示

| 命令 | 说明 |
|------|------|
| `/dashboard` | 打开全屏仪表盘 |
| `/dashboard bar` | 切换底部状态栏显示 |
| `/log off\|last\|all` | 日志面板模式 |
| `/compact` | 手动压缩当前任务的上下文 |

### 3.5 系统命令

| 命令 | 说明 |
|------|------|
| `/config` | 打开系统设置面板 |
| `/manual` | 打开使用手册（HTML） |
| `/upgrade` | 检查并升级 OpenDesk |
| `/notification` | 打开通知中心 |
| `/btw <问题>` | 快速提问（不使用上下文和工具） |

### 3.6 记忆命令

| 命令 | 说明 |
|------|------|
| `/memory add [提示]` | 从当前对话提取记忆 |
| `/memory search <查询>` | 搜索相关记忆并注入上下文 |
| `/memory forget <描述>` | 删除匹配的记忆 |
| `/memory list` | 列出所有记忆 |

详见 [6.5 记忆系统](#65-记忆系统)。

### 3.7 撤销与恢复

| 命令 | 说明 |
|------|------|
| `/undo [N]` | 回退最近 N 条消息，默认 1 条 |
| `/redo [N]` | 恢复最近 N 次回退，默认 1 次 |

### 3.8 调试命令

| 命令 | 说明 |
|------|------|
| `/debug log [on\|off]` | 调试日志开关 |
| `/debug taskid` | 显示当前任务 ID |

详见 [6.3 调试与日志](#63-调试与日志)。

### 3.9 配置口令

| 命令 | 说明 |
|------|------|
| `/quickpass config base` | 导出基础设置口令（不含应用配置） |
| `/quickpass config all` | 导出全部设置口令 |

详见 [快速开始 - 配置分享](/quickstart#share-config)。

## 5. 任务与会话管理

### 5.1 任务状态

| 状态 | 说明 |
|------|------|
| `idle` | 空闲，等待输入 |
| `running` | 运行中，处理请求 |
| `completed` | 已完成 |
| `error` | 错误 |

### 5.2 消息队列

当 AI 正在处理时输入的新消息会进入队列，依次执行。使用 `/cancel` 可清空队列。

### 5.3 上下文压缩

当对话过长时，使用 `/compact` 压缩上下文，减少 Token 消耗。

## 6. 高级特性

本章节介绍 OpenDesk CLI 的高级功能，适合需要深度定制和自动化的用户。

<a id="61-无头模式-headless"></a>
### 6.1 无头模式 (Headless)
无头模式不启动 TUI，适合脚本化和自动化场景。

**基本用法：**

```bash
opendesk -c "你的命令" [选项]
```

**输出格式：**

- **[用户]**：用户输入（青色）
- **[助手]**：AI 回复（绿色）
- **[思考]**：推理过程（灰色）
- **[工具调用]**：工具调用卡片
- **[错误]**：错误信息（红色）

**执行统计：**

```
────────────────────────────────────────────────────────────────────────────────
任务完成
  执行时间: 1m 23.5s
  Token 消耗: 12500 (prompt: 3200, completion: 9300)
────────────────────────────────────────────────────────────────────────────────
```

**权限处理：** 无头模式下，所有权限请求自动拒绝并输出警告日志。

**示例：**

```bash
# 简单任务
opendesk -c "Hello World 的 Go 语言实现"

# 带导出
opendesk -c "分析这个日志文件" --export-html analysis.html

# CI/CD 集成
opendesk /path/to/project -c "代码审查" --model gpt-4 --export-json result.json
```

<a id="62-插件系统"></a>
### 6.2 插件系统
OpenDesk 使用插件架构扩展功能。

**CLI 命令：**

```bash
# 安装插件 (npm包名、路径或 .tgz 文件 URL)
opendesk plugins install <spec>

# 卸载插件
opendesk plugins uninstall <插件ID>

# 列出插件
opendesk plugins list

# 启用插件
opendesk plugins enable <插件ID>

# 禁用插件
opendesk plugins disable <插件ID>
```

**列表输出格式：**

```
已安装的插件:
  my-plugin (npm:@example/my-plugin) [✓ enabled] installed: 2024-01-15
  another-plugin (path:./plugins/another) [✗ disabled] installed: 2024-01-10
```

<a id="64-工具管理"></a>
### 6.4 工具管理
**列出所有工具：**

```bash
opendesk tools list
```

输出示例：

```
[filesystem]
  read_file    - 读取文件内容
  write_file   - 写入文件内容
  list_dir     - 列出目录内容

[search]
  web_search   - 网络搜索
  bing_search  - Bing 搜索
```

**查看工具详情：**

```bash
opendesk tools info <工具名>
```

显示工具名称、描述、所属工具集和输入参数 Schema。

**直接调用工具：**

```bash
# 无参数调用
opendesk tools call list_dir

# 带参数调用
opendesk tools call read_file '{"path": "./README.md"}'
```

<a id="65-记忆系统"></a>
### 6.5 记忆系统
OpenDesk 的记忆系统支持跨会话持久化信息存储。

**记忆类型：**

- **全局记忆 (global)**：所有工作空间共享
- **工作空间记忆 (workspace)**：仅当前工作空间可见

**子命令：**

| 命令 | 说明 |
|------|------|
| `/memory add [提示]` | 从对话中提取有价值的信息保存到长期记忆 |
| `/memory search <查询>` | 搜索相关记忆并注入上下文 |
| `/memory forget <描述>` | 删除匹配的记忆 |
| `/memory list` | 列出所有记忆 |

**使用示例：**

```bash
/memory add                                    # 自动提取
/memory add 请记住我偏好使用 TypeScript        # 带提示提取
/memory search 我的项目                        # 搜索并注入
/memory forget 关于Python的项目                # 删除匹配项
/memory list                                  # 查看所有记忆
```

AI 会自动判断每条记忆应保存到全局还是工作空间。

<a id="66-技能系统"></a>
### 6.6 技能系统 (Skills)
技能是封装好的复杂工作流程，可复用和分享。

**内置技能：**

- 邮件处理
- 日历管理
- 搜索集成
- 文件管理
- 终端操作

**安装第三方技能：**

直接告诉 OpenDesk 安装即可：

```

```

**导出技能：**

```bash
/export skill <技能名> <路径.zip>
```

安装后的技能会自动注册为斜杠命令。

## 常见问题

### Q: 首次启动显示"模型未配置"

输入 `/config`，依次配置：

1. 模型服务商配置 → 添加服务商
2. 模型配置 → 添加模型
3. 默认模型配置 → 设为默认

详见 [快速开始 - 初始配置](/quickstart#initial-config)。

### Q: 命令执行后无响应

- 等待 AI 处理完成
- 使用 `/stop` 停止后重试
- 检查网络连接
- 使用 `/debug log on` 查看日志

### Q: 无法安装插件

- 检查 npm 源是否可访问
- 确认插件包名正确
- 确保有写权限

### Q: 导出文件失败

- 检查指定路径是否存在
- 确认有写权限
- 确保磁盘空间充足
