# 2026.5.5 日构建更新内容

## 新增功能

- 实现多 Agent 框架和注册机制，并支持在子任务创建时按照任务类型自动匹配适合的Agent;
- 实现 Explore Agent 以及 Plan Agent, 重写 Plan Mode 以使用这两个 Agent 来完成代码仓探索和计划编制的任务，降低整体上下文开销，提升规划模式的效果;
- (Cli) 支持在 /config 设置页面中为模式切换，模型切换等基础功能配置自定义快捷键;
- (桌面端) 支持在发送按钮处配置使用 Enter 发送或使用 Ctrl+Enter 发送消息;
- (桌面端) 新增任务自动化应用，支持周期任务，定时任务，按需触发任务的图形化创建，支持一系列常见任务模板以降低使用门槛;

## 优化与 Bug 修复

- 修复 OpenClaw 插件兼容层，适配 5.3 版本的 OpenClaw Plugin SDK，以修复新版微信插件无法运行的问题;
- 优化权限系统，暂时不再提示所有跟命令执行相关的权限以减少误报;
- 优化文件IO访问性能，消除同步文件读写以减少多任务并行执行过程中的卡顿问题;
- 优化微信和飞书等渠道触发任务的任务命名可读性;
- 重构读文件工具 ReadFileTool, 支持同时按行和按大小进行部分读操作, 并在返回结果中加入行号以提升模型理解能力;
- (Cli) 在Cli中不再进行任何Markdown的html渲染，提升TUI响应速度;
- (Cli) 修复 /skillName 方式加载时触发的skill时工具显示不完整的问题，以及与DeepSeek-V4系列的消息格式适配问题;
- (桌面端) 添加设置向导页面，在没有配置默认模型的时候会自动显示;
- (桌面端) 优化日志显示页面，支持日志筛选和列表显示，优化日志系统性能;
- (桌面端) 重写技能广场后端，不再使用cli获取技能，避免在HarmonyOS设备上当前部分Skill市场cli无法运行的问题, 并优化技能广场整体页面体验;
- (桌面端) 重写日历应用，使用更加一致的UI界面设计语言，支持日历项的拖拽等一系列新功能，并支持在Agent创建日历条目时使用与日历应用一致的嵌入卡片显示;
- (桌面端) 优化左侧侧边栏的显示效果，采用更加一致的视觉风格，并优化单个任务下多子任务的显示;
- (桌面端) 支持 Ctrl+{+,-}缩放，Ctrl+0重置缩放;
- (桌面端) 支持npm和pip使用独立于家目录的全局环境;
- (桌面端) 优化日志页面，Markdown渲染页面，大量工具的渲染样式，并统一制定渲染风格设计说明书供后续新增工具参考;
